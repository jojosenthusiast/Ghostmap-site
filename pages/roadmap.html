Roadmap

V1 is shipping.
V2 is the vision.

GhostMap V1 is a focused, free tool for any developer. V2 adds intelligence, collaboration, and team features — built on the same model, not a different product.

V1 — Available now

The foundation.

Everything in V1 is free and open source. It works for any developer, on any codebase, without an account.

Ghost Tree shipped

Live symbol map of any file. Classes, functions, interfaces — organized by real code structure, not folder names.

Ghost Ranges shipped

Named code sections that own everything inside. The cleanest way to mark a refactor, a migration, or a hot zone.

Ghost Anchors shipped

Contextual (attaches to nearest symbol) and named (independent node). Both carry description and status.

Ghost Index shipped

Loading engine that persists the tree between sessions. Open a file: the tree is there instantly.

Ghost Engine shipped

Language extraction stack: LSP → Tree-sitter WASM → regex fallback. 19 languages. Always produces a tree.

Diagnostics shipped

Unclosed ranges, detached anchors, invalid status, malformed syntax — caught as you type with one-click fixes.

V2 — Vision

The next layer.

V2 builds on V1's model. The same Ghost Tree, the same vocabulary — extended with intelligence and collaboration. V1 Core stays free. V2 introduces an Enterprise tier for teams.

Ghost Comments invisible annotations

In V1, @ghost annotations live as comment lines in your source code. In V2, they move out of the file entirely — stored in the Ghost Index — and VS Code renders them as invisible overlays. Your code stays clean. A migration engine converts existing V1 annotations automatically.

Ghost Threads code discussions

Conversations tied to a specific block of code. Not a global chat — a focused thread attached to exactly the function, class, or Ghost Range it's about. Decisions, open questions, architectural rationale — where they belong. Slack context flows in automatically; GhostMap surfaces reminders when related code is touched.

Ghost Graph code flow

Visual maps tracing how code flows: a variable declared here, used there, passed to this function, returned from that one. Ghost Graph follows the path from definition to last usage — across files — and renders it as a navigable graph. Built for understanding unfamiliar code fast.

Agile integration enterprise

Status changes in GhostMap flow to Jira (and other trackers). Sprint timelines surface in the Ghost Tree as urgency signals. When a review passes, GhostMap commits automatically and updates the relevant anchor — the developer doesn't have to push anything manually. Scrum workflow built into the tool, not bolted on top.

AI descriptions & suggestions enterprise

Descriptions generated automatically for undocumented symbols. GhostMap reads the code, writes the annotation. Suggestions for Ghost Ranges on dense files. Context-aware reminders based on what's in Ghost Threads and what's pending in the tree.

V1 → V2 Migration

Your V1 annotations are safe.

When V2 ships, GhostMap will include a migration engine that converts existing @ghost comment lines into invisible Ghost Comments stored in the Ghost Index. The conversion is automatic. Your code gets cleaner; your annotations stay intact.

The V2 Ghost Engine will be a superset of V1 — same syntax, same vocabulary, same tree model. Every concept you learn today carries forward.
